#　git pull时报错

大意就是说你修改的文件在之前的commit中没有提交，远端的文件和本地的文件冲突
解决方法

  1. 昨天遇到这个问题的时候是本地修改的文件不需要提交了，直接放弃就可以了，

    然后远端的文件就覆盖了你的本地文件，然后在git pull,之后就可以git add、git commit、git push三部曲了

    1. 还可以保存本地的状态
      git stash
      然后再
      git pull origin master
      但是提示我No local changes to save

然后我想放弃这两个文件就报错了
